## 4. Sync Flow (End-to-End)

1. **Startup**: `App.xaml.cs` → `IAppLifecycleSyncService.Start()`
2. **Ticker**: 30s loop in `AppLifecycleSyncService` checks `SyncTrigger` + single-flight gate
3. **Orchestration**: `SyncCoordinator` is called:
   - **Order servers** via `ServerSelectionPolicy` (demoted → end, config order preserved)
   - **Resolve protocol** via `NetworkTimeProtocolClientResolver` (SNTP / RFC-868)
   - **Query time** via `SntpClient` or `Rfc868Client`
   - **Apply policy** via `TimeAdjustmentPolicy` (AutoAdjust / AskUser / WithinThreshold / Unavailable)
4. **Persist**: `ISyncStateRepository` + `ILogRepository`
5. **Notify UI**: `DashboardRefreshNotifier` fires event
6. **Render**: `Dashboard.razor` queries `INetworkTimeWorkspaceService.GetDashboardSnapshotAsync()`

---

## 5. Settings Flow

1. **Edit**: User edits in `Settings.razor` → binds to `SettingsEditorModel`
2. **Reorder**: Pointer-based drag via `settings-drag.js` → `SettingsEditorModel.MoveServer()`
3. **Validate**: Validation grouped under "Time Servers" section
4. **Persist**: `ISettingsRepository.SaveAsync()` → `JsonFileStore<AppSettingsSnapshot>`
5. **Consume**: Core services read from repository as needed

**Constraints:**

- Exactly 5 server slots (`SettingsEditorModel.ServerCount`)
- Reorder order = sync priority order

---

## 6. Dashboard Data Flow

1. **Compute**: `SyncCoordinator` builds `DashboardSnapshot` after each sync
2. **Snapshot includes**: `ServerStatusSnapshot[]`, `ServerHealthState`, sync offsets, round-trip delays
3. **Notify**: `DashboardRefreshNotifier` signals UI
4. **Query**: UI calls `GetDashboardSnapshotAsync()`
5. **Render**: Dashboard updates via Blazor data binding + local ticking
